- git remote set-url origin ...: Поменять ssh-версию на https-версию и обратно. (если еще не настраивал ssh-ключи для связи своей машины и битбакет аккаунта - самое время это сделать)

- git remote show origin: Посмотреть, откуда сделан клон (куда сейчас смотрит origin).

- git fetch: Проверить, что изменилось на сервере.

- git pull: Накатить изменения на локальную версию из сервера.

- git log -5: Посмотреть историю последних 5 коммитов.

- git whatchanged -5: Посмотреть, какие именно файлы изменялись в последних 5 коммитах.

- git checkout -b new: Ответвиться из мастера в ветку с любимым именем по твоему выбору.

- git push origin new: Закинуть эту ветку на сервер.

- git checkout master; git branch -D new: Переключиться обратно на мастер, удалить ветку локально.

- git checkout -b new origin/new: Переключиться снова в эту ветку, создав ее копию с сервера.

- git push origin :new: Удалить ее на сервере.

- git checkout master; git branch -d new: Переключиться на мастер, условно удалить ветку локально (с проверкой, присутствуют ли все коммиты этой ветки в мастере).

- git reset --hard HEAD~6; git chechout -b new; Срезать 6 верхних коммитов мастера (локально!), ответить полученное состояние в новую ветку, закинуть ее на сервер.

- git rebase -i HEAD~3; squash: В свежесозданной ветке сжать последние 3 коммита в 1.

- git cherry-pick master: Сверху положить верхний коммит из мастера.

- git reset --soft HEAD~1: Развернуть его в изменения. Закоммитить с другим именем.

- git commit --amend -m "1111": Снова изменить его имя, не разворачивая в изменения.

- git push origin new: Залить все полученное на сервер.

- git checkout <hash>: Откатиться к коммиту.
